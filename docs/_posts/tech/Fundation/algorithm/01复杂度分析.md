## 什么是复杂度分析？

1. 数据结构和算法本身解决的是如何让代码运行得更快，让代码更省存储空间。因此需从执行时间和占用空间两个维度来评估数据结构和算法的性能(执行效率)。

2. 分别用时间复杂度和空间复杂度两个概念来描述性能问题，二者统称为复杂度。

3. 复杂度描述的是算法执行时间（或占用空间）与数据规模的增长关系。


## 为什么要进行复杂度分析？

1. 通过统计、监控代码的运行，虽然就能得到算法执行的时间和占用的内存大小。**但是这种事后统计法测试结果非常依赖测试环境，并且测试结果受数据规模的影响很大**。

2. 和性能测试相比，大O复杂度表示法不用具体的测试数据来测试，就可以粗略地估计算法的执行效率的方法。

3. 掌握复杂度分析，将能编写出性能更优的代码，有利于降低系统开发和维护成本。


## 如何进行复杂度分析？

> 只关注循环执行次数最多的一段代码

| Big O nation |      name      |
| :----------: | :------------: |
|     O(1)     |   常数复杂度   |
|   O(log n)   |   对数复杂度   |
|     O(n)     | 线性时间复杂度 |
|    O(n^2)    |      平方      |
|    O(n^3)    |      立方      |
|    O(k^n)    |      指数      |
|    O(n!)     |      阶乘      |

其中 `BigOLog` 需要多说的是：对数阶时间复杂度非常常见，同时也是最难分析的一种时间复杂度。

```js {2}
// O(log n)
function BigOLog(n) {
    for(let i = 1; i < n; i *= 2) { // important -> i *= 2
        console.log(i)
    }
}
```
第2行代码是循环执行次数最多的。

从代码中可以看出，变量 i 的值从 1 开始取，每循环一次就乘以 2。

实际上，变量 i 的取值就是一个等比数列。如果我把它一个一个列出来，就应该是这个样子的：

::: tip 等比数列
2^0 2^1 2^2 2^3 ....2^k...2^x = n
:::

只要知道 x 值是多少，就知道这行代码执行的次数了。

通过 2x=n 求解, x=log2n，所以，这段代码的时间复杂度就是 O(log2n)。

并且对数之间是可以互相转换的，log3n 就等于 log32 * log2n，所以 O(log3n) = O(C *  log2n)，其中 C=log32 是一个常量。

而在采用大O标记复杂度的时候，可以忽略系数，即 O(Cf(n)) = O(f(n))。所以，O(log2n) 就等于 O(log3n)。

因此，在对数阶时间复杂度的表示方法里，我们忽略对数的“底”，统一表示为 O(logn)。


### 大O表示法的特点

以时间复杂度为例，由于时间复杂度描述的是算法执行时间与数据规模的增长变化趋势，所以常量阶、低阶以及系数实际上对这种增长趋势不产决定性影响，所以在做时间复杂度分析时忽略这些项。

### 复杂度分析法则

1. 单段代码看高频：比如循环。

2. 多段代码取最大：比如一段代码中有单循环和多重循环，那么取多重循环的复杂度。

3. 嵌套代码求乘积：比如递归、多重循环等

4. 多个规模求加法：比如方法有两个参数控制两个循环的次数，那么这时就取二者复杂度相加。

```js
// 嵌套代码求乘积 O(n^2)
function cal(n) {
    let r = 0;
    
    for(let i = 0; i < n; i++) {
        r += f(i)
    }

    return r
}

function func(n) {
    let sum = 0;
    for(let i = 0; i < n; i++) {
        sum += i;
    }
    return sum    
}
```
