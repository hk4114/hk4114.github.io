### 垃圾回收机制

>JS中垃圾回收机制中有两种方式：标记清除和引用计数

1. 标记清除
在函数中生命一个变量，将该变量标记为"进入环境"，当变量离开环境时，则将其标记为"离开环境"。

2. 引用计数（不常见）
跟踪记录每个值得被引用的次数。当声明了一个变量并将一个引用类型值赋给该变量时，则这个值得引用次数就是1 ，如果同一个值又给另一个变量，则该值的应用次数+1。相反，如果包含对这个值引用的变量又取得了另外一个值，则这个值得引用次数-1.当这个值的引用次数变成0时，则说明没有办法再访问这个值了。当垃圾收集器下次再运行时，他就会释放那些引用次数为0的值的内存。但是有一个严重的问题:循环引用。
离开作用域就被垃圾回收了,但是依照引用计数则不会被回收,因为他们的引用次数永远不会为0。
```js
function problem(){
	var obj1 = new Object();
	var obj2 = new Object();
	
	obj1.someOtherObject = obj2;
	obj2.anotherObject = obj1
}
```
3. 性能问题
优化内存占用的最佳方式，就是为执行中的代码只保存必要的数据，一旦数据不再有用，最好通过将其值设置为null来释放其引用，*试用于大多数全局变量和全局对象的属性*。

### 作用域
>作用域有：全局作用域，函数作用域和块级作用域(ES6)。

每次进入一个新的执行环境，都会创建一个用于搜索变量和函数的作用域链。
函数的局部环境不仅有权访问函数作用域中的变量，而且有权访问包含其的父环境，乃至全局作用域。

1. 延长作用域
>在作用域链的前端临时增加一个变量对象，该变量会在代码执行后被移除。
   - try-catch语句的catch块。
   - with语句。

### 闭包和块作用域
```
//点击弹出下标
		var aBtn = document.getElementsByTagName('input');
//闭包写法
		for(var i = 0;i<aBtn.length;i++){
			(
				function(i){ //接受闭包的(i)
					aBtn[i].onclick = function(){
						alert(i)
					}
				}
			)(i)//传参 for循环的i
		}
 //使用块级作用域后
		for(let i = 0;i< aBtn.length;i++){
			aBtn[i].onclick = function(){
				alert(i)
			}
		}
```
